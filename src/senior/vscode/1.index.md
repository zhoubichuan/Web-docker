# 项目开发

::: tip 前言
有些特殊场景下可以将项目制作成镜像，启动容器，然后 vscode 支持进入容器内部开发代码，开发完成后提交到远程仓库
:::

场景：

- 多个不同项目，依赖的环境不同（如 node 有老版本、也有新版本的总之一个系统环境中不能运行多个项目，或者要频繁配置环境）
- 多人协同开发一个项目，依赖频繁变动，需要经常重新安装依赖，电脑安装依赖很耗时间
- 远程开发项目
